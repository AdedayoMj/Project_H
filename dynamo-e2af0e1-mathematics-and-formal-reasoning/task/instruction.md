A combinatorial chemistry group is scoping a substitution library built on macrocyclic ring scaffolds. `/app/input/ring_recipes.json` has a `recipes` array; each entry has a `recipe_id`, `n_positions` (the number of substitution sites spaced evenly around the closed ring), `symmetry_group` (either `cyclic` for rotation-only identification or `dihedral` for rotation-plus-reflection identification), `composition` (an object mapping each substituent code to the exact number of units of it that must be installed — the values sum to `n_positions`, reflecting a fixed feedstock: every unit gets used, none are left out or substituted), and `forbidden_adjacent_pairs` (a list of two-element lists of substituent codes; a steric clash table — the two substituents named in a listed pair may never be placed on two ring positions that are physically adjacent, i.e. positions `i` and `i+1` for any `i`, where position `n_positions - 1` and position `0` also count as adjacent). A listed pair may name the same substituent code twice (for example `["Me", "Me"]`), which forbids placing two units of that substituent on adjacent positions.

Two substitution patterns are the same physical molecule if they are related by a symmetry allowed by that recipe's `symmetry_group`: `cyclic` recipes identify rotations only, while `dihedral` recipes also identify flipped versions of the ring. Some scaffolds are chiral and some are achiral, so do not assume one global symmetry rule across the file. For each recipe, compute the exact number of physically distinct substitution patterns that use every unit of `composition` and contain no forbidden adjacent pair anywhere around the ring, counting each such physically-distinct pattern once regardless of how many of its allowed symmetries would otherwise look like separate raw assignments.

Do not modify anything under `/app/input/`. Write `/app/output.json`, a single JSON object with exactly one key, `isomer_counts`, mapping every `recipe_id` from the input to its exact count, given as a JSON integer (not a string, not an approximation).
